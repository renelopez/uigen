## Architecture Overview

UIGen is a Next.js 15 app that lets users describe React components in a chat interface, which are then generated by Claude (via Vercel AI SDK) and displayed in a live preview.

### Core Data Flow

1. User sends a message in `ChatInterface` → `ChatContext` (using `useChat` from `@ai-sdk/react`) → `POST /api/chat`
2. The API route calls Claude (`claude-haiku-4-5`) with two tools: `str_replace_editor` and `file_manager`
3. Claude's tool calls are streamed back to the client and handled by `FileSystemContext.handleToolCall()`
4. `FileSystemContext` updates the in-memory `VirtualFileSystem`
5. `PreviewFrame` watches `refreshTrigger`, transpiles files via Babel, builds an import map with blob URLs, and renders everything inside a sandboxed `<iframe>`

### Virtual File System (`src/lib/file-system.ts`)

All generated code lives in a `VirtualFileSystem` class (in-memory `Map<string, FileNode>`). Nothing is written to disk. The VFS is serialized to JSON when sending chat messages and when saving to the database.

### AI Tools

- **`str_replace_editor`** (`src/lib/tools/str-replace.ts`): Claude's primary tool — supports `view`, `create`, `str_replace`, and `insert` commands
- **`file_manager`** (`src/lib/tools/file-manager.ts`): Handles `rename` and `delete` operations

### Preview Pipeline (`src/lib/transform/jsx-transformer.ts`)

- Transforms JSX/TSX with `@babel/standalone` (runs client-side in the browser)
- Creates blob URLs for each file and assembles them into a native ES module `importmap`
- Third-party packages are automatically resolved from `esm.sh`
- The `@/` alias maps to the VFS root (`/`)
- The preview HTML is injected into an iframe via `srcdoc` with Tailwind loaded from CDN

### Generated Component Conventions (enforced by the system prompt)

- Every project must have `/App.jsx` as the root entrypoint with a default export
- Style with Tailwind CSS only (no inline styles)
- Local imports use the `@/` alias (e.g., `@/components/Button`)

### Authentication (`src/lib/auth.ts`)

JWT-based sessions stored in httpOnly cookies (7-day expiry). Uses `jose` for signing/verification. The middleware at `src/middleware.ts` protects `/api/projects` and `/api/filesystem` routes.

### Database (`prisma/schema.prisma`)

SQLite via Prisma. Two models: `User` and `Project`. Projects store their entire chat history (`messages`) and VFS state (`data`) as JSON strings. The Prisma client is generated into `src/generated/prisma/`.

Reference `prisma/schema.prisma` anytime you need to understand the structure of data stored in the database.

### No-API-Key Mode

If `ANTHROPIC_API_KEY` is absent, `getLanguageModel()` (`src/lib/provider.ts`) returns a `MockLanguageModel` that generates static Counter/Form/Card components using hardcoded responses. `maxSteps` is reduced to 4 in mock mode.

### State Management

Two React contexts wrap the app:
- `FileSystemProvider` — owns the `VirtualFileSystem` instance and exposes file operations
- `ChatProvider` — wraps Vercel AI SDK's `useChat`, wires tool calls to `FileSystemContext.handleToolCall()`
